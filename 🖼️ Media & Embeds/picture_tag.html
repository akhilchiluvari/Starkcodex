<picture> - Responsive Images

üß† What it does:

  • Contains zero or more `<source>` elements and one `<img>` element to offer alternative image versions for different display scenarios.
  • The browser will choose the best match among the `<source>` elements based on screen size, pixel density, or image format.
  • Provides a fallback image in the `<img>` tag for browsers that do not support the `<picture>` element or if no other source matches.

üï∞Ô∏è When to use:

  • For responsive designs where you need to display different images on different devices or screen sizes.
  • When you need to use newer image formats like AVIF or WebP while providing a fallback for browsers that don't support them.
  • To "art-direct" an image, which means showing a cropped or modified version of an image to better fit a specific layout.

üíª Syntax:

<picture>
    <source media="(min-width: 650px)" srcset="../assets/img_wide.jpg">
    <source media="(min-width: 465px)" srcset="../assets/img_medium.jpg">
    <img src="../assets/img_small.jpg" alt="A plate of food." style="width:auto;">
</picture>

üîç Output:

Resize your browser window to see the image change based on the viewport width.
(Requires three local images: `img_wide.jpg`, `img_medium.jpg`, and `img_small.jpg` in an `assets` folder one level up.)
